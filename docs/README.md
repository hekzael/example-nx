# Documentacion

## Objetivo

Este directorio organiza la documentacion del proyecto por dominios y responsabilidades para facilitar el mantenimiento y la navegacion.

## Estructura

- `docs/PRODUCT_ENGINEERING.md`: indice general
- `docs/product.md`: vision, problema y propuesta de valor
- `docs/requirements.md`: alcance y requisitos (FR/NFR)
- `docs/domain.md`: lenguaje ubicuo, modelo y casos de uso
- `docs/architecture.md`: arquitectura y ADRs
- `docs/api.md`: overview y endpoints
- `docs/security.md`: modelo de seguridad
- `docs/data.md`: modelo de datos
- `docs/operations.md`: operación, testing, planning y change management
- `docs/references.md`: referencias cruzadas`n- `docs/backend-structure.md`: estructura de backend (propuesta)`n- `docs/frontend-structure.md`: estructura de frontend (propuesta)`n- `docs/vertical-slice.md`: ejemplo end-to-end`n- `docs/vertical-slice-admin.md`: ejemplo administracion`n- `docs/build-plan.md`: plan de construccion

## Convenciones

- Usar encabezados claros y consistentes.
- Mantener la fecha de actualizacion en documentos de vision o alcance.
- Registrar decisiones en `docs/architecture.md` (seccion ADRs).
- Referenciar otros documentos con rutas relativas.
